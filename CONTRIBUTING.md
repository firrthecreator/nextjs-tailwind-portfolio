# Contributing Guide

Guidelines for contributing to the portfolio project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/firrthecreator/nextjs-tailwind-portfolio.git`
3. Create a feature branch: `git checkout -b feature/amazing-feature`
4. Make your changes
5. Commit: `git commit -m 'Add amazing feature'`
6. Push: `git push origin feature/amazing-feature`
7. Open a Pull Request

## Code Style

- Follow the existing code style
- Use TypeScript for all new code
- Write JSDoc comments for functions
- Use Tailwind CSS for styling
- Run linter: `npm run lint`

## Commit Messages

Use clear, descriptive commit messages:

```
feat: Add new section component
fix: Resolve mobile navigation alignment
docs: Update deployment documentation
style: Format code with Prettier
```

## Pull Request Process

1. Update documentation if needed
2. Add tests if applicable
3. Ensure build passes: `npm run build`
4. Provide description of changes
5. Link related issues
6. Request review from maintainers

## Code of Conduct

- Be respectful and inclusive
- Welcome diverse perspectives
- Help others learn and grow
- Report issues professionally
